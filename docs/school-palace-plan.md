# School Palace Plan

## 1. Product Vision

Build a school-specific Memory Palace for migrant children aged 6–12 at **CEFR A1–A2 level**. Every item is concrete, visual, and immediately useful in the first month of school.

> **Core principle:** School situation → concrete image → ~15 vocab pins → English meaning → multilingual labels → model audit → approved app content.

> **Subroom rule:** One image per subroom. ~15 vocabulary items per image. No giant rooms with 100+ scattered words.

---

## 2. A1/A2 Constraints

### What A1/A2 means here

| Level | What it looks like | What it EXCLUDES |
|-------|-------------------|------------------|
| **A1** | Single nouns, colours, numbers, basic imperatives ("Sit down"), yes/no | Abstract verbs, subjunctive, conditionals, passive voice |
| **A2** | Simple present/past, "Can I...?", "I have...", "I need...", short dialogues (2–3 turns) | Complex clauses, idioms, metaphor, irony |

### Content rules

#### Image vocabulary (pinned to objects)
- **Concrete objects and visible actions only.** You must be able to point to it in the photo.
- **Single words where possible.** pencil, ruler, scissors, book, door, window.
- **No abstract social words here.** "help", "sorry", "please" live in dialogues, not as pins.

#### Dialogue vocabulary (social / functional / emotional)
- **Abstract words live here.** help, sorry, please, thank you, I don't understand, yes, no.
- **Phrases max 6 words.** "Please raise your hand" = 4 words. Good.
- **Simple questions for survival.** "Can I go to the bathroom?" "Where is my bag?" "I am sick."
- **Imperatives for authority.** "Sit down." "Open your book." "Line up."
- **No slang until A2+.** Playground language stays clean and universal.

## 4. English Source Rules (A1/A2)

### Vocabulary rules

| Rule | Good (A1) | Bad (too hard) |
|------|-----------|----------------|
| Concrete only | pencil, board, sandwich | equipment, administration, nutrition |
| Single words where possible | book, pen, door | exercise book, writing instrument, entrance |
| Child's perspective | my bag, your desk | the student's belongings, the pupil's workstation |
| High frequency | water, eat, help | beverage, consume, assist |

### Phrase rules

| Rule | Good | Bad |
|------|------|-----|
| Max 6 words | "Please sit down." (3) | "Would you be so kind as to take your seat" (11) |
| Imperative for commands | "Open your book." | "The students shall open their textbooks" |
| Simple questions | "Can I go?" | "Would it be permissible for me to leave?" |
| Statements for needs | "I am sick." | "I am experiencing symptoms of illness" |

### Dialogue rules (A2 only)

- **2–3 turns maximum** per mini-dialogue
- **One new phrase per dialogue** (reinforce, don't overload)
- **Real situation, not textbook:** Teacher says "Line up, please." Student says "Where?" Teacher points: "Here."

## 5. Language Layer Rules

### Translation principle

English source is the anchor. Target languages are **audited translations**, not independent creations.

### Per-language matrix

| Language | Draft source | Auditor | TTS | Confidence | Notes |
|----------|-------------|---------|-----|------------|-------|
| Italian (IT) | OPUS-MT en-it | `minerva:7b` | Browser voices | High | Strong native fluency |
| French (FR) | OPUS-MT en-fr | `mistral` | Browser voices | High | Watch school register (tu/vous) |
| Maltese (MT) | Manual curation | Human/native | None | Medium | Vocab only, no dialogues |

### Maltese constraints

- **No LLM generation.** Manual curation only.
- **Vocabulary layer only.** Image + MT word label.
- **No phrases or dialogues** until native speaker validation.
- **No TTS.** Visual read-only.

## 6. Image Generation Pipeline

### Principle

Every subroom image is generated **from the vocabulary list**. The prompt lists the ~15 items. The generated image contains those items. Vocabulary pins are placed on the actual objects.

This eliminates the current problem where subroom images don't align with vocabulary.

### Prompt structure

```
A {angle} view of a {scene} in a warm children's book illustration style.

Visible items: {list of 10-15 items, comma-separated}.

Style: soft colours, friendly, clear outlines, European school setting,
no text, no labels, no watermarks.
```

### Example: "On your desk"

```
A top-down view of a primary school child's wooden desk.

Visible items: yellow pencil, blue pen, pink eraser, wooden ruler,
red notebook, green textbook, zip pencil case, pencil sharpener,
child-safe scissors, glue stick, white paper, printed worksheet,
yellow folder, pink highlighter, box of coloured pencils.

Style: warm children's book illustration, soft colours, friendly,
clear outlines, no text, no labels.
```

### Pipeline

```
1. Define 15 vocabulary items for the subroom
        ↓
2. Write image prompt listing those exact items
        ↓
3. Generate image (ComfyUI/Kimi)
        ↓
4. Review image: are all 15 items visible?
        ↓
5. Write vocabulary data with pin positions (x, y) mapped to actual locations
        ↓
6. Translate to IT, FR, MT
        ↓
7. Model audit (Minerva / Mistral)
        ↓
8. Store approved data
```

### Pin position rule

Pin positions are set **after** the image is generated, not before. The image determines where pins go. Each pin sits on its object.

## 7. Model Audit Pipeline

### Models are auditors, not generators

They check: naturalness, classroom register, grammar, child-appropriateness.

### Audit prompt

```
You are auditing a school-language phrase for children aged 6–12 (CEFR A1–A2).

English source:
"{english}"

Target language: {language}

Candidate:
"{candidate}"

Check:
1. Is it grammatically correct?
2. Is it natural in a classroom?
3. Would a teacher or student actually say this?
4. Is the tone appropriate for A1/A2?
5. Is it 6 words or fewer (if a phrase)?
6. Better version if needed.

Return exactly:
STATUS: PASS | FIX | REJECT
PREFERRED_VERSION: {version}
REASON: {one sentence}
```

### Model assignments

| Language | Model | Role |
|----------|-------|------|
| Italian | `minerva:7b` (Ollama) | Naturalness, register, grammar, classroom realism |
| French | `mistral` (Ollama) | Native French phrasing, school register (tu vs vous), tone |
| Maltese | None | Human/native validation only |

### Audit workflow

```
English source
    ↓
Draft translation (OPUS-MT or manual)
    ↓
Model audit (Minerva / Mistral)
    ↓
├─ PASS → Store, mark audited
├─ FIX  → Apply preferred version, re-audit
└─ REJECT → Discard, new draft
```

## 7. Italian Audit: Minerva

- **Model:** `minerva:7b` via Ollama
- **Validates:** Verb conjugation, article/gender agreement, register, naturalness vs textbook Italian
- **Script:** `scripts/audit-italian.py`
- **Known issue:** May echo template format; parser extracts actual status only

---

## 8. French Audit: Mistral

- **Model:** `mistral` via Ollama
- **Validates:** Natural French phrasing, school register, gender agreement, liaison
- **Key rule:** Teacher → student = "vous" (formal). Student → teacher = "vous". Student → student = "tu" (informal).
- **Script:** To create: `scripts/audit-french.py`

---

## 9. Maltese Limitations

- No LLM generation. No automated translation.
- Manual curation of 200–300 core words.
- Visual dictionary mode only: image + MT label.
- No browser TTS. Native audio recordings if resources allow.

## Appendix B: Audit Checklist

Every subroom module must pass:

- [ ] All English items are A1/A2 level
- [ ] All items are concrete / pointable
- [ ] All phrases are ≤6 words
- [ ] ~15 items per subroom
- [ ] Italian translations audited by Minerva (PASS only)
- [ ] French translations audited by Mistral (PASS only)
- [ ] Maltese content manually curated
- [ ] All dialogues have 2–3 turns max
- [ ] Subroom image exists and is consistent in style
- [ ] TTS works for IT and FR
- [ ] Direction toggle works (EN→X / X→EN)
